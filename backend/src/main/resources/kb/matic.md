# Polygon (MATIC / POL)

## Identity
Polygon is a set of Ethereum scaling solutions launched in 2019 (originally the Matic Network), founded by Jaynti Kanani, Sandeep Nailwal, and Anurag Arjun. Its best-known component is the Polygon PoS chain, an EVM-compatible network that processes transactions cheaply and posts checkpoints to Ethereum. The ecosystem also includes zero-knowledge rollup technology (Polygon zkEVM) and a framework for connecting many chains. The network's native token was MATIC and has been rebranded to POL (the migration began in late 2024 as part of the "Polygon 2.0" upgrade), with POL acting as the ecosystem's gas and staking token.

## Consensus
The Polygon PoS chain uses a Proof-of-Stake system with a set of validators who stake the native token to produce and validate blocks, organized around a Heimdall (validator/checkpoint) and Bor (block-producer) layer. Validators periodically commit checkpoints of the chain's state to the Ethereum mainnet, anchoring Polygon's security to Ethereum. The zkEVM component instead uses validity (zero-knowledge) proofs to inherit Ethereum security.

## Supply schedule
MATIC launched with a maximum supply of 10 billion tokens. As part of the rebrand, MATIC is being migrated 1:1 to POL. POL's design introduces ongoing emissions intended to fund validator staking rewards and ecosystem development, so unlike fixed-supply MATIC the POL model allows for a low, continued issuance rate. Circulating supply reflects vesting, staking, and the ongoing MATIC-to-POL migration.

## Use case
Polygon is used to scale Ethereum: it provides low-cost, fast transactions for DeFi, NFTs, gaming, and payments while remaining EVM-compatible so existing Ethereum applications can deploy with minimal changes. The native token is used to pay gas fees and to stake for securing the PoS chain. Polygon's broader stack aims to connect multiple chains and rollups for shared liquidity and interoperability.

## Key risks
Risks include the operational complexity of the MATIC-to-POL migration and the Polygon 2.0 transition; intense competition from other Ethereum layer-2 rollups (optimistic and zk); validator concentration on the PoS chain; reliance on Ethereum for final settlement; smart-contract and bridge risk; and general token and staking regulatory uncertainty.

## On-chain footprint
The Polygon PoS chain is account-based and EVM-compatible. Tracked metrics include daily transactions, active addresses, gas fees, total value locked in DeFi, the amount of the native token staked and validator counts, and checkpoint activity to Ethereum. High transaction counts reflect its low-fee design.
